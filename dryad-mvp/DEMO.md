## 0. 준비 (시연 직전)

```bash
# 개발 서버 시작
cd dryad-mvp
npm run dev   # http://localhost:5173

# 데이터 신선도 확인 (선택)
node scripts/validate.mjs
```

브라우저: **http://localhost:5173**

준비 항목:
- ✅ 화면 1920×1080 권장 (좌측 패널·중앙 지도·우측 상세 3분할)
- ✅ 우측 상세 패널 초기 사이트가 자동 선택돼 있는지 확인 (최고 위험도 사이트)
- ✅ 시연용 CSV (`public/samples/`) 다운로드 위치 미리 확인
- ✅ 헤더 "데이터 신선도" 배지가 "방금" 또는 "X시간 전"으로 표시되는지

## 2. 통합 현황 — 좌측 패널 (30초)

**시연 동작:**
1. 좌측 상단 "**경기 통합 현황**" 영역 확인
2. 5개 카드 (전체·즉시점검·주간점검·산불연계·예상 관리비용·연간 탄소흡수)

**멘트:**
> "총 6,918개 사이트. 기본 상태에서 A등급(즉시 점검)은 0개, B등급(단기) 7개입니다. 이건 정직한 분포예요 — **경기는 산불·홍수 위험이 낮은 시도**라 종합점수가 36~61에 압축돼 있습니다.
>
> 시연 임팩트는 잠시 후 demo CSV 업로드로 자연스럽게 보여드릴 거예요. 등급 임계값은 기획서 9.2 그대로(80/60/40) 사용하여 정직성을 유지합니다."

## 3. 시군별 분석 — 히트맵 탭 (60초)

**시연 동작:**
1. 중앙 상단 **"시군별 분석"** 탭 클릭
2. 31×6 매트릭스가 펼쳐짐 → 상위 시군 (양평·화성·남양주) 강조 색상
3. **양평군 행 클릭** → 자동 필터링

**멘트:**
> "**산림청 산불피해대장 10년 실측 1,250건**을 시군별로 분해하면 양평이 12.8건/년으로 최다입니다. 시도 평균 일률 적용보다 **시군별 변별력**이 명확히 드러납니다.
>
> 자치구 단위 데이터(수원 장안, 성남 분당 등)는 시 단위로 자동 합산되어 31개 시군 모두에 적용됩니다."

## 4. 사이트 검색 + 상세 패널 (90초)

**시연 동작:**
1. 헤더 검색창에 "**용문**" 입력 → 양평 용문산 진입로 가로수 표시
2. 사이트 클릭 → 우측 상세 패널 열림
3. **3개 섹션 순서대로 시연:**
   - **기본정보**: 좌표·관리기관·연간 탄소흡수량
   - **추가 분석 데이터** ⭐ (새로 추가됨): 수관폐쇄율 · SDR · 도시숲표준지 등급 · VWorld PNU 등 12종 메타
   - **위험도 구성**: 6요인 막대 그래프 + 데이터 출처 배지 (real/proxy/simulation)

**멘트 (추가 분석 데이터 강조):**
> "**경기기후플랫폼 통합으로 사이트별 정밀 데이터를 확보했습니다.**
> - 수관 폐쇄율 X.X% (위성 + LiDAR 기반)
> - 시군 홍수위험 지수 (산림청 산불통계 60% + 경기기후 홍수 40% 가중)
> - 산사태 1등급/취약 여부 (해당 시 자동 가산)
> - SDR(과밀 지표): 본수/거리 — 과밀 시 식생 취약성 +가산
> - VWorld 정밀 PNU 19자리 (6,898/6,918 사이트, 99.7%)
>
> 모든 데이터의 출처가 우측 배지로 표시되어 정직성을 확보합니다."

## 5. 산식 공개 — 산식·출처 모달 (40초)

**시연 동작:**
1. 헤더 우측 "**산식·출처 보기**" 버튼 클릭
2. 6요인 카드 중 **피해 이력** 클릭 → 시군별 실측 산식 보여줌
3. 식생·수종 클릭 → 4단계 우선순위 산식

**멘트:**
> "모든 산식과 임계값은 인앱 모달과 SCORING.md에 100% 공개됩니다.
> - 피해이력 = 산림청 산불 60% + 경기기후 홍수 40% (시군별 실측)
> - 식생 = 도시숲표준지 활력도 > 수관폐쇄율 + 수종 매트릭스 > 수종 매트릭스 + SDR 가산
> - 등급 임계값은 기획서 9.2 그대로(80/60/40) 적용 — 보정 없이 정직한 분포 유지
>
> 심사·감사 시 산식 검증 자료로 그대로 활용 가능합니다."

## 6. PDF 보고서 생성 (40초)

**시연 동작:**
1. 우측 상세 패널 상단 "**📄 PDF 보고서**" 버튼 클릭
2. 1~2초 후 다운로드 시작
3. 다운로드된 PDF 열어서 6개 섹션 확인

**멘트:**
> "사이트 단위 PDF 보고서가 1클릭으로 생성됩니다. 한국어 폰트(Pretendard) 임베드. 6개 섹션:
> 1. 대상 정보 (관리기관·좌표·연간 탄소흡수)
> 2. 위험도 평가 (종합점수 + 6요인 막대)
> 3. **추가 분석 데이터** (수관폐쇄율·SDR·도시숲표준지·VWorld PNU)
> 4. AI 추천 조치 + 예상 비용
> 5. 관리이력 (사용자 입력)
> 6. 민원·신고 (CSV 업로드 매칭 결과)
>
> 회의 자료·결재 문서로 즉시 활용 가능합니다."

## 7. CSV 업로드 시연 (50초)

**시연 동작 — 사이트 CSV:**
1. 헤더 "+ 사이트 CSV" 클릭
2. `demo_streets_gyeonggi.csv` 다운로드 → 업로드
3. 미리보기 → "12개 추가" 클릭
4. 새 가로수길이 지도 + 우선순위 테이블에 표시

**시연 동작 — 민원 CSV:**
1. 헤더 "+ 민원 데이터" 클릭
2. `demo_complaints_gyeonggi.csv` 다운로드 → 업로드
3. "50건 매칭" 결과 → 주입
4. 우선순위 정렬을 "민원 많은 순"으로 변경
5. 민원 누적된 사이트 클릭 → 상세 패널에 **"원점수 X.X + 민원 가산 Y점, 등급 C → B"** 표시

**멘트:**
> "지자체 담당자는 새올·신문고에서 export한 민원 CSV를 그대로 올리면 됩니다. 좌표 기반 **500m 이내 가장 가까운 사이트에 자동 귀속**되고, 누적 민원은 위험도 가산점(최대 +25)으로 반영됩니다. 등급이 한 단계 올라가는 것이 즉시 시각화됩니다."
