# 시연용 샘플 CSV

대시보드의 `+ 사이트 CSV` / `+ 민원 데이터` 업로드 기능을 시연·검증할 때 사용하는 가짜 데이터입니다.

## 파일 목록

### 사이트 추가용

| 파일 | 내용 | 업로드 방법 |
|---|---|---|
| **demo_streets_gyeonggi.csv** | 경기 12개 가로수길 (수원 화성행궁로·일산 호수공원 등) | 헤더 `+ 사이트 CSV` 버튼 |

포함 사이트 — 경기 주요 시군의 대표 가로수길:
- 수원 화성행궁로 왕벚나무 (수원)
- 일산 호수공원 메타세쿼이아 (고양)
- 분당 탄천변 벚꽃길 (성남)
- 파주 임진강변 곰솔 (파주)
- 용인 에버랜드 진입로 단풍나무 (용인)
- 가평 잣향기푸른숲 진입로 (가평)
- 포천 국립수목원 진입로 노송 (포천)
- 화성 우정사람관광 해안로 곰솔 (화성)
- 안성 미리내 진입로 은행나무 (안성)
- 남양주 다산 정약용 유적길 (남양주)
- 양평 두물머리 진입로 (양평)
- 여주 신륵사 진입로 단풍나무 (여주)

업로드 시 형식: 경기도 가로수길정보표준데이터 (시점·종점 좌표·수종·식재년도)

### 민원 데이터 (새올·신문고 export 가정)

| 파일 | 건수 | 시군 커버리지 | 누적 효과 시연 지점 |
|---|---|---|---|
| **demo_complaints_gyeonggi.csv** | 50건 | 31/31 시군 | 수원 행궁동 4건, 고양 일산/덕양 4건, 성남 분당 4건, 용인 3건, 가평·포천·연천 산불우려 |

업로드 시 형식: 새올행정시스템·국민신문고 export 형식
- 컬럼: `민원ID, 시군, 주소, 위도, 경도, 접수일자, 민원유형, 처리상태, 내용`
- 좌표 기반 500m 이내 가장 가까운 사이트에 자동 매칭

## ⚠️ 정직성

이 파일들은 **시연 효과를 위해 손으로 작성한 가짜 데이터**입니다. 실제 시민이 제기한 민원·실재 공공데이터셋 등재 사이트가 아닙니다.

실 운영에선:
- 사이트 CSV → 공무원이 다운로드받은 표준데이터 또는 자체 수목 대장
- 민원 CSV → 새올·국민신문고에서 export한 실제 민원

## 시연 흐름 (권장)

1. `+ 사이트 CSV` → `demo_streets_gyeonggi.csv` 업로드 → 12개 추가, 검색에 "화성행궁" 입력하면 표시됨
2. `+ 민원 데이터` → `demo_complaints_gyeonggi.csv` 업로드 → 50건 매칭
3. 우선순위 테이블 정렬 "민원 많은 순" → 수원 행궁동·고양 일산·성남 분당 상단
4. 화성행궁로 가로수길 사이트 클릭 → 상세 패널에 "민원 4건 · 가산 +10점 · 등급 변동" 확인
5. PDF 보고서 다운로드 → A4 1~2페이지 산출
